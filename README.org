* readability-metric
Readability metrics for EPUB books.

** Nix usage

Process a directory of EPUB files to JSON Lines:

#+begin_src shell
nix run . -- --corpus-dir ~/Downloads --output results/downloads-readability.jsonl --jobs 4
#+end_src

Run the test suite:

#+begin_src shell
nix develop -c make test
#+end_src

Build the packaged command:

#+begin_src shell
nix build
#+end_src

** Python usage

#+begin_src shell
python -m venv .venv
. .venv/bin/activate
python -m pip install -r requirements.txt
python -m readability_metric --corpus-dir ~/Downloads --output results/downloads-readability.jsonl
#+end_src

** Output

The default batch processor writes one JSON object per line. Each record includes EPUB metadata, detected language, token counts, vocabulary-growth fit parameters when the book is below the configured size limit, and per-file errors without stopping the whole batch.

For ranking by lexical difficulty, prefer =predicted_unique_words_20k= over the raw slope. It estimates how many unique word forms the model expects in the first 20,000 words, using both the fit slope and intercept. Lower values mean slower vocabulary growth and therefore easier books by this metric.

#+begin_src shell
jq -r '
  select(.language=="ca" and .status=="ok" and .predicted_unique_words_20k != null)
  | [.predicted_unique_words_20k, .word_count, .unique_words, (.creator // ""), (.title // "")]
  | @tsv
' results/downloads-readability.jsonl | sort -n | head -20
#+end_src

The legacy MongoDB path is still available by passing =--database= with =--book= or =--corpus-dir=.
